# 《凤凰架构》3.1.2 实现隔离性 学习实践计划

## 学习目标
通过逐步实践，理解事务隔离性的实现机制，包括：
1. 事务隔离级别的概念与区别
2. 锁机制（排他锁、共享锁）
3. MVCC（多版本并发控制）
4. 乐观锁实现

## 环境准备
- MySQL 8.0+
- InnoDB存储引擎
- 两个终端窗口模拟并发事务

## 实践步骤

### 步骤1：基础环境搭建
- 创建测试数据库和表
- 理解事务的基本操作

### 步骤2：事务隔离级别演示
- READ UNCOMMITTED（读未提交）
- READ COMMITTED（读已提交）
- REPEATABLE READ（可重复读）
- SERIALIZABLE（串行化）

### 步骤3：锁机制实践
- 排他锁（X-Lock）
- 共享锁（S-Lock）
- 行锁与表锁

### 步骤4：MVCC原理演示
- 多版本数据的读取
- 版本链与可见性规则

### 步骤5：乐观锁实现
- 基于版本号的乐观锁
- 冲突检测与处理

## 实践脚本说明
每个步骤对应一个或多个SQL脚本，通过两个并发终端执行，观察不同隔离级别下的行为差异。
